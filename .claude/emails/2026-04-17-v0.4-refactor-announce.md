## What's new in v0.4 (and usable now)

**Class-based conduct model API.** `Bertrand`, `Cournot`, `Monopoly`, `PerfectCompetition`, `MixCournotBertrand`, `Vertical` are now proper classes with clean constructors. No more string dispatch via `ModelFormulation`.

```python
from pyRVtest import Problem, Bertrand, Cournot, Vertical, Monopoly
models = [
    Bertrand(ownership='firm_id'),
    Cournot(ownership='firm_id'),
    Vertical(downstream=Bertrand(ownership='firm_id'),
             upstream=Monopoly(ownership='manufacturer_id')),
]
problem = Problem(..., models=models)
```

**Dearing et al. (2026) simple-markup models.** Rule-of-thumb and constant-markup now ship as first-class classes. `RuleOfThumb` is an ergonomic wrapper over the existing `cost_scaling` machinery (no new math); `ConstantMarkup` introduces a new additive-markup path.

```python
from pyRVtest import RuleOfThumb, ConstantMarkup
models = [
    RuleOfThumb(phi=2.0),        # p = 2 * mc  (Escudero 2018, "Keystone")
    RuleOfThumb(phi=1.5),        # p = 1.5 * mc
    ConstantMarkup(markup=0.50), # Δ_jt = $0.50 (Example 7)
]
```

**Problem-level taxes with salience opt-out.** Taxes now live on `Problem` (matching the DGP vs. candidate-model conceptual split); individual models can opt out for salience testing. Cleaner for the common case, and the salience testing pattern is explicit.

```python
problem = Problem(
    ...,
    unit_tax='tax_col',
    advalorem_tax='vat_col',
    advalorem_payer='consumer',
    models=[
        Bertrand(ownership='firm_id'),                            # tax is salient
        Bertrand(ownership='firm_id', unit_tax_salient=False),    # salience test
    ],
)
```

**Known-coefficient cost shifters on Formulation.** For non-tax cost components with known pass-through coefficients (Dearing's general framework):

```python
cost_formulation = Formulation('0 + w', known_coefficients={'input_price': 0.75})
```

**Labor-side conduct testing (skeleton).** `Problem(market_side='labor')` with `Monopsony`, `BertrandWages`, `CournotEmployment` — real sign-flipped markup math. `NashBargaining` imports but raises `NotImplementedError` pending the labor paper formula. The full `LaborSupplyBackend` compute_jacobian / compute_hessian arrive in v0.5 when labor project data is available. **We'd appreciate a sign-convention sanity check** — see ask below.

**Pass-through diagnostics on `ProblemResults`.** Implements the Dearing et al. (2026) Remark 4 distinguishability condition:

```python
df = results.passthrough_comparison(metric='offdiag_frobenius')
# DataFrame: market_id × (unordered model pair) × distance
```

Currently requires all candidate models to be `Vertical` (the existing `build_passthrough` restriction). Non-Vertical pass-through lands in v0.5.

**Friendly results exports.**

```python
results.to_dataframe()         # long-form pairwise RV / F / MCS
results.summary_df(alpha=0.05) # wide-form with reject flags
results.to_latex(path=..., caption=..., label=...)
results.to_markdown()
```

**Panel-of-problems aggregation.** For AFSSZ-style multi-market-year workflows:

```python
from pyRVtest import PanelResults
panel = PanelResults(results={(market, year): pr for ... in ...})
panel.rejection_rates(alpha=0.05)
panel.summary_df()
panel.to_dataframe()
```
