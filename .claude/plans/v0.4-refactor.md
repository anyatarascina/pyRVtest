# pyRVtest v0.4 Refactor — Design Document

**Status:** Draft for coauthor review
**Target branch:** `v0.4-refactor` (off CClean-fixes)
**Last updated:** 2026-04-16

This is a living design document. Decisions go in the Decisions Log section at the bottom; open questions go in Open Questions. Do not re-litigate things in Decisions Log without a compelling reason.

---

## 1. Motivation

The CClean-fixes branch shipped 12 correctness fixes, a 900x clustering speedup, and the `demand_params` feature. In the process, three structural issues became unavoidable:

1. **`problem.py` is 1,855 lines** with a single `Problem` class at ~1,435 lines. It does data validation, markup computation, IV correction, orthogonalization, demand adjustment (two parallel paths), test statistics, MCS simulation, and output assembly.

2. **PyBLP internal coupling is fragile.** `_compute_demand_adjustment_gradient` directly mutates `demand_results._sigma`, `._pi`, `._beta`, `._rho`, `._delta` and calls the public `compute_delta()` to rebuild state. Any PyBLP rename of these private attributes breaks us silently. CClean-fixes wrapped the mutation in try/finally but did not remove the coupling.

3. **Two parallel demand paths.** The `if self.demand_params is not None` branch appears in 5+ places. Two ~200-line methods (`_compute_analytical_demand_adjustment`, `_compute_demand_adjustment_gradient`) return identical output shapes from completely separate code. Adding a labor supply system or a user-supplied demand system would add a third branch in each place.

Three forthcoming applications all stress these issues:
- **AFSSZ scalable testing:** 910 market-years, needs clean batch/aggregation API, benefits from analytical Hessian.
- **Labor market conduct (basic_model.tex):** wants labor supply primitives, sign-flipped pass-through, monopsony model types.
- **Custom demand systems:** researchers with their own demand code need a documented extension point.

A single abstraction (DemandBackend, with implementations for PyBLP, logit/nested logit, user-supplied, and labor supply) resolves all three. This refactor makes the AFSSZ deployment cleaner and sets up the labor deployment for free.

## 2. Goals

Must achieve (v0.4 full scope). Grouped by category:

### Architecture

1. **Encapsulate PyBLP coupling.** All access to `pyblp.ProblemResults` private attributes flows through a single class.
2. **Split `DemandBackend` into two protocols.** Core `DemandBackend` (Jacobian, Hessian) and optional `SupportsDemandAdjustment` mixin (xi, Z_D, W_D, xi_gradient, jacobian_gradient). `UserSuppliedBackend` can implement only the first; the testing engine checks `isinstance(backend, SupportsDemandAdjustment)` before requesting first-stage correction inputs. Cleaner contract, honest about what each backend provides.
3. **Single dispatch path for demand.** No `if demand_params is not None` in user-facing code. Single code path generic over backend.
4. **Split `Problem` into staged pipeline.** Each stage (markups, orthogonalize, demand adjustment, test engine) is a standalone module with clear inputs/outputs.
5. **Class-based `ConductModel` API (Option B).** Each model is a class (`Bertrand`, `Cournot`, `ConstantMarkup`, `RuleOfThumb`, `NashBargaining`, etc.) with its own `compute_markup()` method. `ModelFormulation` kept as a thin backward-compat alias that constructs the right class. Removes string dispatch in `evaluate_first_order_conditions`.
6. **Results aggregation for panels.** `ProblemResults.to_dataframe()`, `to_latex()`, `summary_df()`, `to_markdown()`, and a `PanelResults` wrapper for many market-years.
7. **Stand-alone `build_passthrough`.** Diagnostic function, not tied to `solve()`.
8. **Instrument construction helpers.** `pyRVtest.instruments` module with both product-side (`rival_sums`, `differentiation_ivs`, `blp_instruments`) and labor-side (`hausman`, `bartik`, `concentration_hhi`) constructors.
9. **Labor-side hooks.** `market_side='labor'` flag, `Monopsony`/`BertrandWages`/`CournotEmployment`/`NashBargaining` model classes, `LaborSupplyBackend` skeleton wired through the protocol.
10. **`UserSuppliedBackend` with worked example.** Not sketched; demonstrably usable with an Almagro-Sood-style labor supply system. Test coverage + documented integration path.

### Correctness

11. **Analytical nested logit Hessian.** Closed-form second derivative, replacing finite-diff currently used by vertical + demand_params.
12. **First-stage correction equivalence test.** Direct test that `demand_adjustment=True` produces the same TRV/F under PyBLP path and demand_params path on matched DGP. Closes the gap identified in §3 note 4.
13. **Snapshot-based regression tests.** Every existing test's TRV/F/MCS/markups/g/Q serialized to JSON; every migration step must reproduce to `atol=1e-10`.
14. **Property-based tests** (Hypothesis library): determinism, row-permutation invariance, market-partition invariance, FWL identity.
15. **Golden-file replication.** One published result (DMSS yogurt recommended) pinned with exact expected TRV/F/MCS. Any deviation triggers explicit review.
16. **`mypy --strict` compliance across the package.** Explicit types everywhere, no `Any`.
17. **Doctest coverage.** Every public function/class has a runnable docstring example. `pytest --doctest-modules` passes in CI.

### Developer / AI experience

18. **Error messages as first-class API.** Rich validation errors stating what was expected, what was received, and the fix. Pydantic-style or equivalent for input validation.
19. **Structured logging.** Replace `print` / `output()` calls with `logging` module at per-module loggers. Configurable levels so users can silence specific subsystems.
20. **Public API pin.** `__all__` in every `__init__.py`; deprecation policy documented; `DeprecationWarning` raised for aliased old-API names (e.g., `sigma` as alias for `rho`).
21. **Documentation for AI usage.** `AGENTS.md` at repo root (contributors), `docs/agent_guide.md` (user-facing, also accessible via `pyRVtest.show_agent_guide()`), `docs/custom_demand.md` (worked-example integration guide).

### Polish

22. **Bug fixes and renames.** `Dict_K` class attribute → instance, `sigma → rho` rename to match PyBLP (deprecation alias for one release).

### Defer to v0.5+

- Full Almagro-Sood labor supply backend with ordered choice sets (v0.4 ships `UserSuppliedBackend` + labor supply skeleton; full implementation when labor project has data).

## 3. Prototype: `PyBLPBackend`

The single highest-risk element of the refactor is encapsulating PyBLP's private-attribute mutation cleanly. Here's a working prototype that answers the question "does encapsulation actually work?"

```python
# backends/base.py
from typing import Protocol, Optional, Hashable, Tuple, List
from contextlib import contextmanager
import numpy as np


class DemandBackend(Protocol):
    """Abstract protocol for demand primitives consumed by pyRVtest.

    A backend provides ds/dp (Jacobian), optionally d2s/dp2 (Hessian),
    and the machinery needed for the demand-adjustment first-stage
    correction in the RV test variance.
    """

    # --- identification ---

    @property
    def n_parameters(self) -> int: ...

    @property
    def theta_names(self) -> List[str]: ...

    # --- primitives (always supported) ---

    def compute_jacobian(
        self,
        market_id: Optional[Hashable] = None,
    ) -> np.ndarray:
        """ds/dp at current parameters. (J, J) for one market, (N, J_max) stacked."""

    def compute_hessian(
        self,
        market_id: Hashable,
    ) -> Optional[np.ndarray]:
        """d2s/dp2 at current parameters. (J, J, J). None if backend doesn't support it."""

    # --- demand adjustment (optional) ---

    def supports_demand_adjustment(self) -> bool: ...

    def demand_moments(self) -> Tuple[np.ndarray, np.ndarray, np.ndarray]:
        """Return (xi, Z_D, W_D) for the psi first-stage correction."""

    def xi_gradient(self) -> np.ndarray:
        """d(xi)/d(theta), shape (N, n_parameters)."""

    def jacobian_gradient(
        self,
        market_id: Hashable,
    ) -> np.ndarray:
        """d(D_jk)/d(theta_i) for one market, shape (J, J, n_parameters)."""

    # --- context manager for perturbation (optional; used by finite-diff fallback) ---

    @contextmanager
    def perturbed(self, theta_index: int, delta: float):
        """Yield self with theta[theta_index] += delta. Must restore on exit."""
```

### `PyBLPBackend` implementation

```python
# backends/pyblp.py
import contextlib
import os
from contextlib import contextmanager
import numpy as np

from .base import DemandBackend


class PyBLPBackend:
    """Wraps a pyblp.ProblemResults object. All private-attribute access lives here.

    The wrapper intentionally hides _sigma, _pi, _beta, _rho, _delta from the rest
    of pyRVtest. If PyBLP renames these, only this file needs updating.
    """

    def __init__(self, demand_results, cache_jacobian: bool = True):
        self._results = demand_results
        self._cache_jacobian = cache_jacobian
        self._jacobian_cache = None
        self._theta_names, self._theta_indices = self._enumerate_theta()

    # --- parameter enumeration ---

    def _enumerate_theta(self):
        """Build flat list of (name, accessor, index) for all non-zero parameters.

        Mirrors the iteration order used in the current PyBLP finite-diff code:
        sigma (flat), then pi (flat), then alpha (beta with label 'prices'), then rho.
        """
        names, indices = [], []
        r = self._results
        K2 = r.problem.K2
        D = r.problem.D

        # sigma
        for (i, j) in np.ndindex(K2, K2):
            if r.sigma[i, j] != 0:
                names.append(f'sigma[{i},{j}]')
                indices.append(('sigma', i, j))

        # pi
        for (i, j) in np.ndindex(K2, D):
            if r.pi[i, j] != 0:
                names.append(f'pi[{i},{j}]')
                indices.append(('pi', i, j))

        # alpha (price coefficient in beta)
        price_idx = [k for k, v in enumerate(r.beta_labels) if v == 'prices']
        for k in price_idx:
            names.append('alpha')
            indices.append(('beta', k, None))

        # rho
        if len(r.rho) > 0:
            for k in range(len(r.rho)):
                names.append(f'rho[{k}]')
                indices.append(('rho', k, None))

        return names, indices

    @property
    def n_parameters(self) -> int:
        return len(self._theta_names)

    @property
    def theta_names(self):
        return list(self._theta_names)

    # --- primitives ---

    def compute_jacobian(self, market_id=None):
        if self._jacobian_cache is None:
            with contextlib.redirect_stdout(open(os.devnull, 'w')):
                self._jacobian_cache = self._results.compute_demand_jacobians()
        full = self._jacobian_cache
        if market_id is None:
            return full
        idx = np.where(self._results.problem.products.market_ids.flatten() == market_id)[0]
        block = full[idx][:, ~np.isnan(full[idx]).all(axis=0)]
        return block

    def compute_hessian(self, market_id):
        with contextlib.redirect_stdout(open(os.devnull, 'w')):
            return self._results.compute_demand_hessians(market_id=market_id)

    # --- demand adjustment ---

    def supports_demand_adjustment(self) -> bool:
        return True

    def demand_moments(self):
        r = self._results
        Z_D = r.problem.products.ZD
        W_D = r.updated_W
        xi = r.xi
        return xi, Z_D, W_D

    def xi_gradient(self):
        r = self._results
        xd_cols = r.problem.products.dtype.fields['X1'][2]
        price_in_linear = 'prices' in xd_cols
        XD = r.problem.products.X1
        if price_in_linear:
            XD = np.delete(XD, xd_cols.index('prices'), 1)
        partial_y_theta = (
            np.append(r.xi_by_theta_jacobian, -r.problem.products.prices, axis=1)
            if price_in_linear else r.xi_by_theta_jacobian
        )
        if r.problem.ED > 0:
            absorbed, _ = r.problem._absorb_demand_ids(partial_y_theta)
            n_theta = self.n_parameters
            partial_y_theta = absorbed.reshape(-1, n_theta)
        if not XD.shape[1]:
            return partial_y_theta
        ZD, WD = r.problem.products.ZD, r.updated_W
        product = (
            XD @ np.linalg.inv(XD.T @ ZD @ WD @ ZD.T @ XD)
            @ (XD.T @ ZD @ WD @ ZD.T @ partial_y_theta)
        )
        return partial_y_theta - product

    def jacobian_gradient(self, market_id):
        """Finite-difference fallback using the perturbed() context manager."""
        eps = _get_eps()
        J_t = self.compute_jacobian(market_id).shape[0]
        grad = np.zeros((J_t, J_t, self.n_parameters))
        for k in range(self.n_parameters):
            with self.perturbed(k, +eps / 2):
                jac_plus = self.compute_jacobian(market_id)
            with self.perturbed(k, -eps / 2):
                jac_minus = self.compute_jacobian(market_id)
            grad[:, :, k] = (jac_plus - jac_minus) / eps
        return grad

    # --- perturbation context manager ---

    @contextmanager
    def perturbed(self, theta_index: int, delta: float):
        """Apply delta to theta[theta_index], invalidate caches, yield, restore."""
        kind, i, j = self._theta_indices[theta_index]
        saved = self._save_state()
        try:
            self._apply_perturbation(kind, i, j, delta)
            self._recompute_delta()
            self._jacobian_cache = None  # invalidate
            yield self
        finally:
            self._restore_state(saved)
            self._jacobian_cache = None

    def _save_state(self):
        r = self._results
        return {
            'sigma': r.sigma.copy(),
            'pi': r.pi.copy(),
            'beta': r.beta.copy(),
            'rho': r.rho.copy() if len(r.rho) != 0 else None,
            'delta': r.delta.copy(),
        }

    def _apply_perturbation(self, kind, i, j, delta):
        r = self._results
        if kind == 'sigma':
            r._sigma[i, j] = r._sigma[i, j] + delta
        elif kind == 'pi':
            r._pi[i, j] = r._pi[i, j] + delta
        elif kind == 'beta':
            r._beta[i] = r._beta[i] + delta
        elif kind == 'rho':
            r._rho[i] = r._rho[i] + delta
        else:
            raise ValueError(f"unknown parameter kind: {kind}")

    def _recompute_delta(self):
        with contextlib.redirect_stdout(open(os.devnull, 'w')):
            self._results._delta = self._results.compute_delta()

    def _restore_state(self, saved):
        r = self._results
        r._sigma[:] = saved['sigma']
        r._pi[:] = saved['pi']
        r._beta[:] = saved['beta']
        if saved['rho'] is not None:
            r._rho[:] = saved['rho']
        r._delta = saved['delta']


def _get_eps():
    from pyRVtest import options
    return options.finite_differences_epsilon
```

### What the prototype reveals

1. **The encapsulation is mechanically straightforward.** All five private attributes are written in `_apply_perturbation` and `_restore_state`; all four reads of `r.sigma`, `r.pi`, `r.beta`, `r.rho` stay inside `PyBLPBackend`. The rest of pyRVtest sees only the `perturbed(k, delta)` context manager and the five public methods.

2. **The abstraction unifies the two paths.** The current `_compute_demand_adjustment_gradient` can become a generic function over any backend that implements `perturbed()` + `compute_jacobian()` + `demand_moments()`:

   ```python
   def compute_demand_adjustment(backend: DemandBackend, problem, M, N, markups, ...):
       eps = options.finite_differences_epsilon
       grad = np.zeros((M, N, backend.n_parameters))
       for k in range(backend.n_parameters):
           with backend.perturbed(k, +eps / 2):
               markups_plus = build_markups(problem, backend, ...)
           with backend.perturbed(k, -eps / 2):
               markups_minus = build_markups(problem, backend, ...)
           grad[:, :, k] = _finite_diff(markups_plus, markups_minus, eps)
       xi, Z_D, W_D = backend.demand_moments()
       partial_xi = backend.xi_gradient()
       H = (1 / N) * Z_D.T @ partial_xi
       h = (1 / N) * Z_D.T @ xi
       h_i = Z_D * xi
       return grad, H.T @ W_D, H, h_i, h
   ```

   The `LogitBackend` can override any of these with closed-form implementations (e.g., `LogitBackend.xi_gradient()` returns the analytical derivative; `LogitBackend.perturbed()` is a cheap swap of `alpha`/`sigma` attributes). The caller writes the same code.

3. **Jacobian caching is a real win.** The current PyBLP path calls `compute_demand_jacobians()` inside each `_compute_markups` call, which happens once per perturbation. Caching at the backend level (invalidated only on `perturbed()`) avoids redundant computation.

4. **Methodology equivalence (resolved via DMSS 2024 Appendix C).** The required object for the RV first-stage correction when demand is estimated is explicitly given by DMSS (Duarte, Magnolfi, Sølvsten, Sullivan 2024, QE) Supplementary Material Appendix C, equation (77):

   $$\tilde\psi_{m,i} = \hat\psi_{m,i} - \hat W^{1/2} G_m \Lambda [h(\hat\theta^D) - h_i(\hat\theta^D)]$$

   where $G_m = -\tfrac{1}{n}\hat z' \nabla_\theta \hat\Delta_m(\hat\theta^D)$ and $\Lambda = (H' W^D H)^{-1}H' W^D$. The gradient $\nabla_\theta \hat\Delta_m$ is exactly d(markup)/d(θ). Both pyRVtest paths target this object:

   - **demand_params analytical path** computes $\partial\hat\Delta_m / \partial\theta$ via implicit differentiation of the FOC at observed shares, using closed-form $\partial D/\partial\theta$.
   - **PyBLP finite-diff path** perturbs θ, calls `compute_delta()` which runs the BLP contraction so $s_\text{impl}(\delta(\theta+\epsilon), \theta+\epsilon) = s_\text{obs}$, then evaluates markup via FOC at observed shares and the Jacobian at the restored $(\delta, \theta+\epsilon)$ point.

   Both approaches compute the same sample function $\hat\Delta_m(\theta^D)$ and its gradient to first order. Agreement to $O(\epsilon^2)$ finite-diff error. **No coauthor adjudication required; the paper adjudicates.** Regression test at migration step 0c provides empirical confirmation.

   Separately, when `endogenous_cost_component` is set, a DMQSS (2026) "Testing with Non-constant Cost" Appendix B correction is also applied (the `(1/2)W^{3/4}(W^+ Z z^r q^e \lambda'_q + \lambda_q z' q^e Z W^+)W^{3/4}g_m` term on p. 52 of the working paper). The memo's "Appendix B of our paper" reference meant DMQSS 2026 Appendix B, not DMSS 2024 (DMSS's supplementary material does not have an Appendix B; the demand-correction is in Appendix C).

5. **One messy bit.** `_theta_indices` has to know the iteration order to expose a flat index space. This is PyBLP-specific. The `LogitBackend` would have a different internal layout. The `perturbed(k, delta)` protocol hides this from callers, but the internal bookkeeping is backend-specific. Acceptable.

**Prototype verdict:** the encapsulation works. The refactor is not speculative; Lorenzo's design maps to concrete code cleanly.

### 4.2 `ConductModel` API — Option B (class-based)

Each conduct model is a class with a `compute_markup()` method. `ModelFormulation` becomes a thin backward-compat alias that routes string-style construction to the right class.

```python
# models/base.py
from typing import Protocol, Optional
import numpy as np


class ConductModel(Protocol):
    """Protocol for a single conduct model.

    A conduct model is responsible for computing implied markups in one market
    given the demand Jacobian, ownership matrix, shares, and prices.
    """

    name: str  # canonical short name, e.g. "bertrand"

    def compute_markup(
        self,
        D: np.ndarray,            # (J, J) demand Jacobian for this market
        O: np.ndarray,            # (J, J) ownership matrix
        s: np.ndarray,            # (J,) market shares
        p: np.ndarray,            # (J,) prices (or wages for labor)
        market_id: Hashable,
        product_data,             # raw market-slice for reading model-specific cols
    ) -> np.ndarray:
        """Return (J,) implied markup vector."""

    def required_columns(self) -> list[str]:
        """Names of product_data columns this model reads (for validation)."""
```

Concrete classes (illustrative — full set in step 5 of migration):

```python
# models/standard.py

class Bertrand(ConductModel):
    name = "bertrand"

    def __init__(self, ownership: str = "firm_ids", kappa: Optional[str] = None):
        self.ownership = ownership
        self.kappa = kappa

    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        return -np.linalg.solve(O * D, s)

    def required_columns(self) -> list[str]:
        cols = [self.ownership]
        if self.kappa is not None:
            cols.append(self.kappa)
        return cols


class Cournot(ConductModel):
    name = "cournot"
    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        return -(O * np.linalg.inv(D)) @ s


class Monopoly(ConductModel):
    name = "monopoly"
    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        return -np.linalg.solve(D, s)


class PerfectCompetition(ConductModel):
    name = "perfect_competition"
    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        return np.zeros_like(s)
```

```python
# models/constant.py

class ConstantMarkup(ConductModel):
    """Markup is a fixed value (per-product or scalar). Rule-of-thumb baseline."""
    name = "constant_markup"

    def __init__(self, markup_value):
        # Either a column name (str) read per-market, or a scalar
        self.markup_value = markup_value

    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        if isinstance(self.markup_value, str):
            return np.asarray(product_data[self.markup_value]).flatten()
        return np.full_like(s, float(self.markup_value))


class RuleOfThumb(ConductModel):
    """Markup is a constant fraction of price: Δ = λ * p."""
    name = "rule_of_thumb"

    def __init__(self, markup_rate):
        self.markup_rate = markup_rate

    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        rate = (
            np.asarray(product_data[self.markup_rate]).flatten()
            if isinstance(self.markup_rate, str) else float(self.markup_rate)
        )
        return rate * p


class CostPlus(ConductModel):
    """Markup is a constant fraction of marginal cost: Δ = λ * c.

    Implementation note: this requires solving Δ = λ*(p - Δ), i.e. Δ = λ*p / (1+λ).
    """
    name = "cost_plus"
    def __init__(self, markup_over_cost):
        self.markup_over_cost = markup_over_cost

    def compute_markup(self, D, O, s, p, market_id, product_data) -> np.ndarray:
        lam = (
            np.asarray(product_data[self.markup_over_cost]).flatten()
            if isinstance(self.markup_over_cost, str) else float(self.markup_over_cost)
        )
        return lam * p / (1.0 + lam)
```

## 10. If refactor fails — rollback procedure

If at any point the v0.4 refactor produces incorrect results (snapshots break unexpectedly, equivalence test fails, replication fails), here is the recovery sequence:

### Soft revert (a single migration step is wrong)
```bash
git checkout v0.4-refactor
git revert HEAD                          # undo the bad commit
# investigate, fix, recommit
```

### Hard revert (multiple steps wrong, want to abandon and restart)
```bash
git checkout v0.4-refactor
git reset --hard <last-good-commit-hash>  # nukes intervening commits, keep branch
# resume from last-good-commit
```

### Nuclear option (abandon v0.4 entirely, return to CClean-fixes)
```bash
git checkout CClean-fixes
git branch -D v0.4-refactor              # delete the refactor branch entirely
# CClean-fixes is the production branch; v0.3.2-stable tag is the anchor
git push origin :v0.4-refactor           # delete remote branch
```

### Verify the recovery
```bash
git log --oneline -5                     # confirm we're at expected commit
pytest                                   # all 66 + step-0 tests should pass
git tag -l                               # v0.3.2-stable should exist
```

### Why this works

- `v0.3.2-stable` tag is a frozen pointer to a known-good commit. Cannot be lost without explicit `git tag -d`.
- `CClean-fixes` carries the step-0 protection (snapshots, equivalence test, golden file). Even if `v0.4-refactor` is destroyed, the protection survives.
- Snapshot tests on `CClean-fixes` mean any later attempt at refactoring (whether v0.4 redo or v0.5) starts with verified baseline outputs.

### What is NOT recoverable

- Manual edits to the snapshot JSON files outside of a deliberate, commit-message-justified update. If snapshots drift silently, we lose the baseline. Discipline: `tests/snapshots/*.json` is read-only except in commits whose message explains the deliberate change and links to the relevant migration step.
- Force-pushed branches. `git push --force` to `CClean-fixes` or main would defeat this. Don't do it. If a coauthor force-pushes, the tag still exists locally but remote history is corrupted.
